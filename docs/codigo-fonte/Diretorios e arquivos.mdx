### 3.1 - Diretórios e Arquivos

No diretório "/" (root) temos a estrutura do projeto e os arquivos de configuração. Nas pastas principais, temos:

<img src="/images/3100.png" />


- **Built Process template** - diretório utilizado para configuração de deploy para Azure

- **Mazetank.SafeTravels** - conteúdo legado sistema anterior

- **SafeTravels.api** - conteúdo principal relacionado a api

- **SafeTravels.Console** - teste de unidade

- **SafeTravels.DB** - diretório legado contendo informações do inicio da plataforma

- **SafeTravels.DataBase** - contexto do banco de dados e biblioteca de classes referente ao banco de dados

- **SafeTravels.Domain** - biblioteca de classes referente ao banco de dados e repositórios

- **SafeTravels.Migrations** - migrações do código para o banco de dados

- **SafeTravels.Util** - Métodos e serviços complementares do sistema

- **SafeTravels.Web** - Painel em MVC

Os arquivos na pasta root:

- **.gitignore** - arquivo com a definição de quais são os pastas/arquivos são intencionalmente deixados de fora (exemplo: pastas src) e não são rastreados.

- **README.md** - arquivo em markdown (. md) com instruções do estrutura do projeto para desenvolvimento.

- **SafeTravels.sln** - arquivo de solução que contém informações baseadas em texto que o ambiente usa para localizar e carregar os parâmetros nome-valor
